# PyTorch 笔记-01

## 模型：

所有的torch模型需要继承nn.Module类

神经元：接收输入、乘上权重、加上偏置，算出总和

$$总和(z)=w⋅x+b$$

启发于人类神经元，在生物学中，神经元收到刺激时，如果刺激不够大，神经元会保持沉默（输出 0）；如果刺激超过某个阈值，神经元就会“放电”激活（输出信号）。激活函数就是在用数学模拟这种“激活”的决策过程。

数学本质：引入非线性，没有激活函数只能是一个简单的线性方程，拟合出各种复杂形状的函数边界

偏置项： 激活函数向左向右平移，让网络学会在什么条件下做出反应

三种最经典的激活函数：

①Sigmoid函数（一个平滑的S形曲线）

- **规则：** 无论输入的数字多大或多小，它都会把结果**强行压缩到 0 到 1 之间**。
    
- **用途：** 比如输入 100，输出可能是 0.99；输入 -100，输出可能是 0.01。它非常适合用来表示**概率**（比如判断这半句话是人话的概率有多大）。早期非常流行。
    

②Tanh双曲正切函数

- **规则：** 和 Sigmoid 类似，但它把结果**压缩到 -1 到 1 之间**。
    
- **特点：** 它的中心点在 0，也就是说它允许输出负数。在很多情况下，它的表现比 Sigmoid 更好。
    

③ReLU函数（Rectified Linear Unit， 线性整流单元）

- **规则极其简单：** 
    
    $$max(0, x)$$
    
    。如果输入是负数，直接输出 0（掐断信号）；如果输入是正数，原样输出。
    
- **地位：** **它是目前深度学习（尤其是图像识别和现代网络）绝对的“王者”和默认选择。** 虽然它看起来就是两根拼接的直线，但组合起来能产生极强的非线性，而且计算速度极快，还解决了很多复杂的数学问题（如梯度消失）。

- Embedding-将原始文本转化为向量形式来方便机器理解

（理想）嵌入技术的两个主要特点：

- **「语义表示」** 某些类型的Embedding可以捕捉单词之间的语义关系。这意味着，含义更接近的单词在向量空间中更接近。例如，“猫”和“狗”的向量肯定比“狗”和“草莓”的向量更相似。
    
- **「维度」** 较小的向量（较低维度）在内存中保存或处理效率更高，而较大的向量（较高维度）可以捕捉复杂的关系，但容易出现过拟合。作为参考，GPT-2 模型系列的嵌入大小至少为 768
    

### 传统embedding：TF-IDF（统计学）

TF-IDF 的理念是通过考虑两个因素来计算单词在文档中的重要性：

- **「词频 (Term Frequency)」** 词语在文档中出现的频率。TF 值越高，表示词语对文档越重要。
    
    *（比如在讲乔布斯的文章里，“苹果”出现了 50 次，TF 很高。）*
    
- **「逆文档频率 (Inverse Document Frequency)」** 词语在文档中的稀缺性。这种方法是基于这样的假设：出现在多篇文档中的词语的重要性低于仅出现在少数文档中的词语。
    
    *（比如“的”、“是”、“在”几乎每篇文章都有，非常不罕见，所以它们的 IDF 接近于 0。而“光刻机”只在少数科技文章出现，IDF 很高。）*
    

**计算公式：**TF-IDF = TF × IDF

输出结果：高维稀疏向量

假设你的词表里有 10 万个词。一篇文章经过 TF-IDF 计算后，会变成一个长达 10 万维的向量：`[0, 0, 0.5, 0, ..., 0.8, 0]`绝大部分位置是 0（因为这篇文章不可能包含词表里所有的词），只有这篇文章出现的词才有大于 0 的分数。

### 静态embedding：Word2Vec（神经网络）

> *“You shall know a word by the company it keeps”*

它不再统计次数，而是通过**训练一个微型神经网络**，让机器学会词与词之间的关系(语义)

两种训练方法：

- **CBOW (连续词袋模型)：** 挖掉一句话中间的词，用两边的词去猜中间的词。
    
    - *例：[我] [爱] ___ [天安门] -> 猜出“北京”。*
        
- **Skip-gram (跳字模型)：** 给你中间的词，去猜它周围可能会出现什么词。
    
    - *例：___ ___ [北京] ___ ___ -> 猜出“我”、“爱”、“天安”、“门”。*
        

word2vec的架构非常简单：首先通过一个**隐藏层**从中提取嵌入，然后外加一个**输出层**，预测词汇表中所有单词的**概率**。表面上，该网络被训练成根据相邻词预测正确的缺失词，但实际上，这是**训练网络隐藏层**并为每个单词找到正确嵌入向量。网络训练完成后，最后一层可以被抛弃，因为找出Embedding向量才是真正的目标。

**工作原理：（CBOW）**

- 选择一个上下文窗口（例如上图上下文窗口为4）。
    
- 将特定单词前后两个单词作为输入。
    
- 将这四个上下文单词编码为one-hot向量。
    
- 将编码后的向量传入隐藏层，该层具有线性激活函数。
    
- 聚合隐藏层的输出（例如，使用 lambda 均值函数）。
    
- 将聚合后的输出输入到最后一层，该层使用 Softmax 预测每个可能单词的概率。
    
- 选择概率最高的 token 作为网络的最终输出
    

特点：能够捕捉词义关系：

`Vector(国王) - Vector(男人) + Vector(女人) ≈ Vector(女王)`

缺陷：无法解决一词多义问题（苹果手机、水果苹果）
